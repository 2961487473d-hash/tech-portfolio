# Quark Audit Hub - AI 模型内容质量自动化标注平台

## 项目概述

**Quark Audit Hub** 是一个基于 AI 大语言模型的自动化内容质量评测系统，专门用于对 AI 模型生成的文本进行多维度、精细化的质量审计和标注。该项目通过双轨并行审计架构（事实核查 + 规则审计），实现了对 AI 生成内容的全方位质量评估。

**项目定位**：面向 AI 模型研发人员、内容质量审核人员、数据标注团队的专业评测工具。

**核心价值**：将传统需要人工逐条审核的内容标注工作自动化，显著提升标注效率和一致性。

## 核心功能

### 1. 双轨并行审计引擎

系统同时运行两个独立的审计流程：

**轨道一：事实核查（Fact Check）**
- 逐句提取内容中的事实性陈述
- 使用权威知识库验证陈述准确性
- 输出：正确/错误/存疑 + 置信度 + 证据来源
- 支持来源链接展示（维基百科、学术论文、官方文档等）

**轨道二：规则审计（Rule Audit）**
- 基于 84 条预定义规则进行内容检查
- 覆盖 7 大维度：内容准确性、需求满足度、合规与人设、信息密度、逻辑组织、用语措辞、特色人感等
- 输出：违规项 + 严重程度 + 置信度 + 违规证据 + 改进建议

### 2. 智能规则匹配系统

**规则结构**：
```typescript
interface AuditRule {
  id: string;              // 规则唯一标识
  dimension: string;       // 所属维度（如"需求满足度"）
  label: string;           // 规则名称（如"潜在意图洞察"）
  keywords: string[];      // 关键词列表
  definition: string;      // 规则定义
  score: string;           // 评分标准
  case: string;            // 典型案例
}
```

**匹配算法**：
1. 关键词正则匹配（快速筛选）
2. 语义理解匹配（AI 深度分析）
3. 多维度权重计算
4. 置信度评估与过滤

### 3. 可视化标注界面

**核心交互**：
- 富文本编辑器：支持分段编辑和实时预览
- 智能高亮：根据规则类型自动着色（事实错误红色、需求满足度蓝色、信息密度紫色等）
- 点击联动：点击审计结果自动定位到原文位置
- 展开/折叠：规则列表支持展开查看详情
- 暗色/亮色模式：支持主题切换

**审计结果展示**：
- 评分卡片：事实核查分数、规则审计分数、问题总数
- 问题列表：按严重程度排序，支持置信度过滤
- 原始输出：可查看模型原始返回内容

### 4. 多模型对比评测

**支持模型**：
- 同时配置最多 4 个模型进行并行评测
- 预设模型配置（SP1/SP2/SP3/SP4）
- 自定义模型供应商和 API Key
- 支持独立配置事实核查模型和规则审计模型

**模型供应商**：
- OpenAI (GPT-4, GPT-4o, GPT-4o-mini)
- Anthropic (Claude-3.5-Sonnet, Claude-3-Opus)
- 智谱 AI (GLM-4, GLM-4-Plus)
- Moonshot (Kimi-K2.5)
- DeepSeek (DeepSeek-V2.5, DeepSeek-Coder)
- 通义千问 (Qwen-Max, Qwen-Plus)
- 硅基流动 (Pro/moonshotai/Kimi-K2.5, Pro/zai-org/GLM-5)
- 自定义供应商

### 5. 规则管理与配置

**84 条预定义规则**（7 大维度）：

| 维度 | 规则数量 | 典型规则 |
|------|---------|---------|
| 需求满足度 | 12 条 | 潜在意图洞察、答非所问、主需遗失、不合理拒答 |
| 内容准确性 | 10 条 | 主需事实错误、对话幻觉、结论有误、过程有误 |
| 合规与人设 | 7 条 | 政治敏感、淫秽色情、涉恐涉暴、违规导流 |
| 信息密度 | 4 条 | 文本精炼建议、信息冗余、关键结论不凸显 |
| 逻辑组织 | 6 条 | 思路混乱、论证链条复杂、结构不合理 |
| 用语措辞 | 5 条 | 语气不当、专业术语滥用、口语化过度 |
| 特色人感 | 8 条 | 共情能力、对话节奏、惊喜感、人设一致性 |

**规则扩展性**：
- 支持通过配置文件动态添加新规则
- 无需修改代码即可扩展规则库
- 规则定义采用结构化 JSON 格式

## 技术挑战与解决方案

### 挑战 1：84 条规则导致提示词过长

**问题**：
- 84 条规则的完整定义约 15,000 tokens
- 超出部分模型的上下文窗口限制
- 请求超时风险增加

**解决方案**：
1. 增加超时时间至 120 秒
2. 使用更强的模型（如 Claude-3.5-Sonnet、GPT-4）
3. 未来优化：实现规则分批处理或预筛选

### 挑战 2：精准定位原文位置

**问题**：
- AI 返回的位置描述不够精确
- 富文本编辑器中存在 HTML 标签
- 需要支持点击联动高亮

**解决方案**：
1. 四级匹配策略（精确位置 → 完整文本 → 关键词 → 强特征）
2. 提取纯文本进行匹配
3. 计算字符偏移量实现精确定位

### 挑战 3：多模型并行调用

**问题**：
- 多个模型同时调用可能失败
- 需要统一错误处理和结果合并
- 配置管理复杂

**解决方案**：
1. Promise.all 并行调用
2. 独立的错误处理机制
3. 结构化配置管理（settings.ts）
